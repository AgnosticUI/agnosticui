# AgnosticUI v2 - LLM Context Guide

## Project Overview
AgnosticUI Local (v2) is a CLI-based UI component library that copies Lit web components directly into your project. Developers own the code, can customize it freely, and avoid dependency lock-in.

## AI Playbooks (Generative UI Patterns)
The project provides structured **AI Playbooks** designed for agentic and generative UI workflows. Each playbook resides in the `/v2/playbooks/` directory and includes prompt files (`PROMPT-3-FRAMEWORKS.md`, `PROMPT-REACT.md`, etc.) that give LLMs the specific context needed to assemble AgnosticUI primitives into complete views.

- **Login Form**: `/v2/playbooks/login/` - Authentication interface patterns.
- **Onboarding Wizard**: `/v2/playbooks/onboarding/` - Multi-step state management and flow.
- **Discovery Dashboard**: `/v2/playbooks/dashboard/` - Layout orchestration for data-heavy views.
- **Support Center**: `/v2/playbooks/support/` - Knowledge base and help desk interaction patterns.
- **Data Grid**: `/v2/playbooks/grid/` - Complex table structures with sorting and filtering logic.
- **Blog**: `/v2/playbooks/blog/` - Content listing and detail page patterns.
- **Landing Page**: `/v2/playbooks/landing/` - Marketing and hero section layouts.
- **Form Association**: `/v2/playbooks/form-association/` - Native form participation with FACE components.

## Key Architecture
- **Framework**: Built with Lit for web components
- **Approach**: CLI-first tool that copies components to your project
- **Language**: Primarily TypeScript (86.8%)
- **Structure**: Core components are located in `/v2/lib/src/components/`
- **React/Vue Wrappers**: Core components are wrapped in React and Vue wrappers for native framework experience (props instead of attributes, proper event handling, full TypeScript support). Located in `/v2/lib/src/components/[COMPONENT]/{react|vue}/` and use barrel files to export their API/interface.

## CLI Usage
The main entry point is `npx agnosticui-cli` (or `npx ag` if installed locally) which copies components directly into target projects. Available commands: `init`, `add`, `remove`, `list`, `sync`, `storybook`, `playbook`, `context` (and deprecated `view`). Run `npx agnosticui-cli --help` for the current list. The CLI source is in `/v2/cli`.

## Component Structure
Each core component is in the `/v2/lib/` - `/v2/lib/src/components/[component-name]/` and contains:
1. Lit web component implementation (.ts)
2. Styles (.css)
3. Tests
4. Documentation (to be added)
5. React and Vue wrappers
6. Barrel files are used for both core and react/vue wrappers.

## Theme System
The project uses a systematic design token approach via Style Dictionary located in `/v2/theme-registry/`:
- **52 production-ready tokens**: colors, spacing, border-radius, surfaces, borders, focus, motion, z-index
- **Prefix**: All CSS variables use `--ag-` prefix (e.g., `--ag-primary`, `--ag-focus`)
- **Output formats**: CSS custom properties, JavaScript exports, JSON
- **Themes**: Light and dark theme variants with automatic token switching via `data-theme` attribute on the `<html>` element (e.g., `[data-theme="dark"]`)
- **White-labeling**: Enterprise-ready theming capabilities
- **Build**: Run `npm run build` in theme-registry to regenerate tokens
- See `/v2/theme-registry/README.md` for complete token documentation

## Project Structure

### Core Library (`/v2/lib/`)
- Component implementations, tests, and framework wrappers
- Dependencies: Lit, TypeScript, @floating-ui/dom, @lit/react, focus-trap
- Check `/v2/lib/package.json` for exact versions

### CLI Tool (`/v2/cli/`)
- Main entry point: `npx agnosticui-cli` command (or `npx ag` if installed locally)
- Copies components directly into target projects
- Run `npx agnosticui-cli --help` for available commands (see `/v2/site/docs/installation.md` for details)

### Documentation (`/v2/site/`)
- Built with Vitepress
- Contains all user-facing documentation and guides
- Component API documentation and usage examples

### Supplementary Docs (`/v2/docs/`)
- Architecture docs, dev guides, SDUI spec, LLM prompt guide
- `llm-prompt-guide.md` - How to instruct LLMs to emit AgNode graphs for the SDUI system
- `sdui-architecture.md` - Full SDUI codegen pipeline and CI architecture

### Skins (`/v2/skins/`)
- `skins-bundle.css` - All named skin themes in one file
- `skin-switcher.css` + `skin-switcher-core.js` - Runtime theme switching utilities
- One subdirectory per named skin (e.g. `autumn-slate/`, `cyberpunk/`, `midnight/`)

### SDUI (`/v2/sdui/`)
- Schema-driven, framework-agnostic UI rendering via `AgNode[]` graphs
- `schema/` - JSON schema (`agnosticui-schema.json`), spec, source, dist; package name `@agnosticui/schema`
- `renderers/` - React, Vue, and Lit `AgDynamicRenderer` implementations
- `demo/`, `demo-vue/`, `demo-lit/` - Framework-specific SDUI demo apps
- `demo-llm/` - Experimental LLM-driven SDUI demo

### Scripts (`/v2/scripts/`)
- `build-local-tarball.sh` - Build and pack CLI as a local tarball for testing
- `generate-playbooks-manifest.mjs` - Regenerate `playbooks-manifest.json`
- `scaffold-component.mjs` - Scaffold a new component skeleton
- `verify-examples.mjs` / `verify-playbooks.mjs` - Smoke-test example and playbook apps

### Test Examples (`/v2/examples/`)
- `/v2/examples/lit-test/` - Lit consumption test artifacts
- `/v2/examples/react-test/` - React consumption test artifacts
- `/v2/examples/vue-test/` - Vue consumption test artifacts
- Real-world integration validation; each has Storybook available via `npx agnosticui-cli storybook`
