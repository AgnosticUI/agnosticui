# AgnosticUI Playbooks

Playbooks are complete, working example applications that demonstrate AgnosticUI components in realistic scenarios. Each playbook provides LLM prompts that can be run with Claude (or similar LLM like gemini-cli, Codex, etc.) to generate implementations for React, Vue, and Lit.

## Directory Structure

```
v2/playbooks/
├── README.md                  # This file
└── login/                     # Login form playbook
    │
    │ # PROMPTS (input - run these with Claude)
    ├── PROMPT-all.md          # Builds all 3 frameworks → all/react/, all/vue/, all/lit/
    ├── PROMPT-react.md        # Builds React only → react/
    ├── PROMPT-vue.md          # Builds Vue only → vue/
    ├── PROMPT-lit.md          # Builds Lit only → lit/
    │
    │ # ASSETS (shared resources referenced by prompts)
    ├── assets/
    │   ├── design/               # Design mockups (Desktop.png, Tablet.png, Phone.png)
    │   ├── logo-3-frameworks.svg # AgnosticUI logo
    │   ├── logo.svg           # Lorumipsum logo
    │   ├── google-icon.svg    # Social login icon
    │   ├── apple-icon.svg       # Social login icon (alternative)
    │   ├── facebook-icon.svg  # Social login icon
    │   └── login-bg.jpg       # Background image
    │
    │ # OUTPUTS (generated by running prompts with Claude)
    ├── all/                   # Output from PROMPT-all.md
    │   ├── react/
    │   ├── vue/
    │   └── lit/
    │
    ├── react/                 # Output from PROMPT-react.md
    ├── vue/                   # Output from PROMPT-vue.md
    └── lit/                   # Output from PROMPT-lit.md
```

## How It Works

Playbooks use a **prompt-driven approach**:

1. **Prompts are the source of truth** - Each `PROMPT-*.md` file contains complete instructions for Claude to build the example
2. **Outputs are generated** - The framework directories (`react/`, `vue/`, `lit/`, `all/`) are created by running prompts with Claude
3. **Assets are shared** - The `assets/` directory contains design files and images that prompts reference and copy into builds

### Building a Single Framework

To build just one framework, run the corresponding prompt with Claude:

```bash
# In the playbook directory, start Claude and reference the prompt
cd v2/playbooks/login
claude # or LLM of choice

# Then ask Claude to follow the prompt
> Please follow the instructions in PROMPT-react.md
```

This generates the example in `react/`.

### Building All Frameworks

To build all three frameworks at once:

```bash
cd v2/playbooks/login
claude # or LLM of choice

> Please follow the instructions in PROMPT-all.md
```

This generates examples in `all/react/`, `all/vue/`, and `all/lit/`.

## StackBlitz Integration

Playbooks are available as live StackBlitz demos on the documentation site. The integration works via the `PlaybookStackBlitz` Vue component which fetches files from GitHub and creates a StackBlitz project on-the-fly.

### Key Files

- **Component:** `v2/site/docs/.vitepress/theme/components/PlaybookStackBlitz.vue`
- **Config:** `v2/site/docs/.vitepress/theme/components/playbooks-config.ts`

### How It Works

1. User clicks "Open in StackBlitz" button on the docs site
2. Component reads the playbook config from `playbooks-config.ts`
3. Files are fetched from `raw.githubusercontent.com`
4. Binary file references (like `.jpg`) are replaced with raw GitHub URLs
5. StackBlitz SDK creates and opens the project

## Creating a New Playbook

### 1. Create the Directory Structure

```bash
mkdir -p v2/playbooks/[name]/assets/design
```

### 2. Add Design Assets

Place shared assets in the `assets/` directory:

- Design mockups in `assets/design/`
- Icons, logos, images directly in `assets/`

### 3. Create the Prompts

Create prompt files that instruct Claude how to build the example:

- `PROMPT-all.md` - Instructions to build all frameworks into `all/react/`, `all/vue/`, `all/lit/`
- `PROMPT-react.md` - Instructions to build React into `react/`
- `PROMPT-vue.md` - Instructions to build Vue into `vue/`
- `PROMPT-lit.md` - Instructions to build Lit into `lit/`

Each prompt should include:

- Project setup instructions (Vite, dependencies)
- AgnosticUI CLI initialization
- Component implementation details
- Asset copying instructions
- Responsive layout specifications

See `login/PROMPT-*.md` for reference.

### 4. Update the StackBlitz Config

Edit `v2/site/docs/.vitepress/theme/components/playbooks-config.ts` to include the new playbook.

### 5. Create the Documentation Page

Add a markdown file at `v2/site/docs/playbooks/[name].md` with the PlaybookStackBlitz component.

### 6. Add to Navigation

Update `v2/site/docs/.vitepress/config.mts` to include the new playbook in the sidebar.

## Binary Files

StackBlitz SDK only supports text files. For binary assets (images, fonts):

1. Keep them in the `assets/` folder of the playbook
2. Prompts copy them to each framework's `public/` folder during build
3. Add URL mappings to `binaryFiles` in the StackBlitz config
