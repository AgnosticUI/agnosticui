# AgnosticUI Project Context

## Repository Structure

All production code lives under `v2/`.

### Core Library
- **Component source:** `v2/lib/src/components/*` - Framework-agnostic web components (Lit)
- **Design tokens:** `v2/theme-registry/` - CSS custom properties, token configs, and theming

### Skins
- **Skin bundle:** `v2/skins/skins-bundle.css` - All skin themes in one file
- **Skin switcher:** `v2/skins/skin-switcher.css` + `skin-switcher-core.js` - Runtime theme switching
- **Named skins:** One subdirectory per skin (e.g. `autumn-slate/`, `cyberpunk/`, `midnight/`)

### Documentation
- **VitePress site:** `v2/site/docs/` - Main docs site (components, playbooks, theming, installation)
- **Supplementary docs:** `v2/docs/` - Architecture docs, dev guides, SDUI spec, LLM prompt guide
- **CLI docs:** `v2/cli/README.md` - Command-line tool documentation

### CLI
- **Source:** `v2/cli/src/` - `agnosticui-cli` npm package
- **Commands:** `add`, `init`, `storybook` (and deprecated `view`)
- **Story generator:** `v2/cli/src/utils/stories.ts`

### Playbooks
- **Location:** `v2/playbooks/[PLAYBOOK]/` - Prompt-driven example apps
- **Completed:** `login`, `onboarding`, `dashboard`, `support`, `blog`, `landing`, `grid`, `form-association`
- **Prompt files:** `PROMPT-REACT.md`, `PROMPT-VUE.md`, `PROMPT-LIT.md`, `PROMPT-3-FRAMEWORKS.md`, `PROMPT-3-FRAMEWORKS-V2.md`
- **Manifest:** `v2/playbooks-manifest.json` (auto-generated by `v2/scripts/generate-playbooks-manifest.mjs`)

### Examples (CLI-generated test apps)
- **Lit:** `v2/examples/lit-test/`
- **React:** `v2/examples/react-test/`
- **Vue:** `v2/examples/vue-test/`

### SDUI (Server-Driven UI)
- **Schema:** `v2/sdui/schema/` - JSON schema (`agnosticui-schema.json`), spec, source, dist
- **Renderers:** `v2/sdui/renderers/lit/`, `renderers/react/`, `renderers/vue/`
- **React demo:** `v2/sdui/demo/`
- **Vue demo:** `v2/sdui/demo-vue/`
- **Lit demo:** `v2/sdui/demo-lit/`
- **LLM demo:** `v2/sdui/demo-llm/` - Experimental LLM-driven SDUI
- **Dev docs:** `v2/sdui/DEV-WORKFLOW.md`, `v2/sdui/PUBLISH.md`

### Scripts
- **Location:** `v2/scripts/`
- `build-local-tarball.sh` - Build and pack CLI as a local tarball for testing
- `generate-playbooks-manifest.mjs` - Regenerate `playbooks-manifest.json`
- `scaffold-component.mjs` - Scaffold a new component skeleton
- `verify-examples.mjs` / `verify-playbooks.mjs` - Smoke-test example and playbook apps
- `bulk-sync.sh`, `setup-testing-playground.sh`, `cleanup-testing-playground.sh`

## Key Principles

- **CSS-first approach:** Components use CSS custom properties for theming
- **Framework-agnostic core:** Web components (Lit) work across React and Vue via wrappers
- **Accessibility-focused:** WCAG 2.1 AA compliance is required
- **Design token system:** Consistent theming via CSS variables from `v2/theme-registry/`
- **CLI-driven workflow:** `agnosticui-cli` (`ag`) is the primary way to add components to user apps

## Common Workflows

### Adding or Updating a Component
1. Edit core in `v2/lib/src/components/[Component]/`
2. Run tests in `v2/lib/`
3. Verify in example apps (`v2/examples/`)
4. Update VitePress docs in `v2/site/docs/components/`

### Documenting a Component
1. Core API in VitePress (`v2/site/docs/components/`)
2. Storybook stories via CLI (`ag storybook`) in example apps
3. Update playbook if component has a specific use case

### Testing the CLI Locally
1. Build tarball: `bash v2/scripts/build-local-tarball.sh`
2. Install into an example app: `npm install ../../cli/agnosticui-cli-X.Y.Z.tgz --force`
3. Run CLI commands against the example app

### SDUI Development
1. Edit schema in `v2/sdui/schema/src/`
2. Build: `cd v2/sdui/schema && npm run build`
3. Update renderers in `v2/sdui/renderers/`
4. Verify demos in `v2/sdui/demo*/`

## Branch Convention
- Feature branches: `issue-[number]/[description]`
- Always branch from `master`
- Never commit directly to `master`
