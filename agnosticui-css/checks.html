CSS Conventions

Just some more copy with another link to inclusive components primarily to test keyboard navigation. Here's with the fieldset borders removed and the legend hidden for sighted users, but still available for assistive technology such as screen readers. This is probably still a poor UX choice, but some designs may require it.

It occurs to me you may wonder why we don't lean more heavily on positional elements in our CSS (e.g. we could do something like [type="radio] + span and similar. This is fine, but since we opt to copy the CSS over to framework implementations like React that use CSS Modules, having a class-driven approach makes that easier. It's also consistent with our loose conformance to Object-Oriented CSS and SMACSS conventions.
